- (decision) Target deliverables: ARCHITECTURE-SPINE.md (developer invariants), High-Level Design (HLD) and a Deployment/Infrastructure Document designed for stakeholders to estimate project budget.
- (decision) Go backend handles real-time telemetry gateway and command controls; Python service handles mission path planning and suggestion calculations.
- (decision) Deploy platform on-premises to support private network security policies and local server hosting.
- (decision) Utilize WebRTC for real-time, low-latency video streaming from the drone/hub to the operator dashboard.
- (decision) Use SRS (Simple Realtime Server) as the WebRTC media gateway for RTSP/RTMP stream ingest and low-latency playout.
- (decision) Use PostgreSQL with PostGIS extension for relational GIS data and Redis for transient telemetry state.
- (decision) Use WebSockets for client telemetry, MQTT for drone/hub communications, and gRPC for Go/Python inter-service calls.
- (decision) Implement a Single-Operator Session Lease in Redis to prevent split-brain command control on active flights.
- (decision) Go gateway acts as geofence gatekeeper, validating path calculations via PostGIS before hub transmission.
- (decision) Archive missions as paired MP4 video and JSON telemetry logs indexed relative to takeoff time (t=0) for client sync playback.
- (decision) Enforce JWT authentication with embedded RBAC claims for all Go HTTP and WebSockets connections.
