## 2. Team Discussion Highlights

### What Went Well?
* **Amelia (Developer):** "Integrating the React Slider with the Map and Video canvas was incredibly clean. Reusing the Leaflet `flightPath` array rendering logic simplified drawing historical trajectories without duplicate code paths."
* **Charlie (Senior Dev):** "Using standard seconds-offset time-indexing (`t=0`) was a great design decision. It made calculating formatted duration timestamps in both Go backend API sorting and React slider labels trivial."
* **Elena (Junior Dev):** "The play/pause auto-increment ticker is extremely responsive. Toggling it updates map markers and canvas text elements synchronously at 1 Hz."

### What Were the Struggles?
* **Dana (QA Engineer):** "If a mission contains thousands of coordinates, reading the entire `missions.json` into memory on the GET API will become a scaling issue. We'll need pagination or chunked loading for production scale."
* **Amelia (Developer):** "We had to carefully clear the replay play/pause interval timer on tabs change and on Replay exit to avoid background loops running indefinitely."
